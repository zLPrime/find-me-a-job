# Template: Application Package

> Structural template only. Every application package requires explicit
> candidate approval before anything in it is sent. See
> [rules/general.md](../rules/general.md).
>
> This template also covers the package *after* submission — status
> progresses through `submitted` rather than ending at `approved`. If
> the candidate applies directly (outside this process) and reports it
> afterward, create or update this artifact retroactively: see
> rules/general.md's "Recording actual submissions" section.

```markdown
# Application Package — <role title> at <employer name>

Status: draft | ready for review | approved | submitted
Last updated: <date>
Linked vacancy: <reference to vacancy artifact>
Linked candidate profile version: <reference/date>

- Summary of tailoring choices: <what was emphasized/reordered and why,

  e.g. a candidate-exported PDF — kept alongside this artifact rather
  than only referencing the source .md draft>

  candidate edits made before sending, so the artifact matches what was
  actually sent, not just the last approved draft>
```
